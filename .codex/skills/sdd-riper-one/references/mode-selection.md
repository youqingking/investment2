# Mode Selection

`sdd-riper-one` 是重型 Harness 教练和控盘器；`sdd-riper-one-light` 是熟练用户的无感安全带。两者不是完整/简化关系，而是职责不同。

## 使用 sdd-riper-one

满足任一条件时，优先使用重型 one：

- 用户、模型或任务还不能稳定切出最小混沌单元。
- 任务目标、边界、上下文、验收方式不清楚。
- 不熟代码库，需要 codemap 降低混沌。
- 需求来自多份文档、会议记录、截图、接口说明或聊天记录，需要 context bundle。
- 涉及安全、权限、资金、数据一致性、状态机、并发时序、跨项目契约、迁移或不可逆操作。
- 需要团队训练、实习生可操作、低质量模型兜底。
- 需要重度留痕、审计、handoff、new chat 恢复或复盘归档。

## 使用 sdd-riper-one-light

满足多数条件时，可以使用 light：

- 用户已经把任务拆得基本可执行。
- 目标、范围、验证方式基本清楚。
- 风险中低，或风险点已由用户明确。
- 模型能力强，用户希望更少阻塞、更快推进。
- 只需要中度留痕、checkpoint、目标复述、进度汇报和验证闭环。

## 从 one 降级到 light

当 spec 已经形成清晰任务单元，且风险可控时，可以建议后续小步执行改用 light：

- 最终目标清楚。
- 当前任务单元可独立验证。
- In / Out 清楚。
- 失败可局部回炉。
- 不再需要频繁问询或方案分叉。

## 从 light 升级到 one

当 light 发现任务明显过大、上下文不足、高风险、连续失败或用户需要训练式引导时，切回 one。
