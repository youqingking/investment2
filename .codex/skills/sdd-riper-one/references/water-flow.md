# Water Flow

水流理论用于判断：模型提出的实现路线和用户原设想不同时，是否应该接受、修正或回炉。

## 基本原则

人给目标、边界、风险闸门和验收证据；模型在边界内按代码地形探索路径。不要要求模型逐字照人脑里的第一条路线走。

## 接受模型新路线的条件

满足多数条件时，可以接受：

- 仍然服务最终目标和当前任务单元。
- 没有越过 In / Out。
- 解释了为什么更贴合当前代码地形。
- 风险更低、改动面更小或验证更清楚。
- 能给出验证证据和失败回炉方案。
- 用户明确接受路线变化，并写入 spec。

## 必须阻塞的条件

- 模型路线改变了需求语义。
- 越过 Scope 或引入无关重构。
- 绕过安全、权限、数据一致性、审计或迁移要求。
- 没有验证证据，却准备宣布完成。
- 连续失败后仍沿同一路径硬跑。

## Spec 记录方式

在 spec 中记录：

```text
Route Change:
- Original assumption:
- Model-proposed route:
- Why it fits code terrain:
- Scope impact:
- Risk:
- Validation:
- User Decision:
```
