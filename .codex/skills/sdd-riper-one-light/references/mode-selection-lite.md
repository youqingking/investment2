# Light Mode Selection

`sdd-riper-one-light` 是熟练用户和强模型的无感安全带。它默认用户已经把任务切到基本可执行的最小混沌单元，只做轻量评估、checkpoint、最小 spec、进度记录和验证闭环。

## 继续使用 light

满足多数条件时，继续 light：

- 用户给出的目标、范围、约束、验证方式基本清楚。
- 任务能在一个局部代码链路、一个功能点或一个明确缺陷内完成。
- 失败可以局部回炉，不会推倒整个任务。
- 模型仍有实现路径选择空间。
- 风险为中低，或风险点已被明确写入 checkpoint。
- 用户希望 agent 自动推进，少打断。

## 升级到 deep

出现以下情况时，在 light 内升级到 `deep`：

- 根因未知，需要更长研究。
- 需要方案比较或风险解释。
- 涉及多个模块但仍在同一项目、同一任务单元内。
- 已经出现一次明显偏差，需要重新对齐目标和方案。

## 切到 sdd-riper-one

出现以下情况时，建议切到 `sdd-riper-one`：

- 用户、模型或任务还不能稳定切出最小混沌单元。
- 需求来自多份文档、会议记录、截图或口述，需要 build context。
- 不熟代码库，需要 codemap 先降混沌。
- 涉及安全、权限、资金、数据一致性、状态机、跨项目契约、迁移或不可逆操作。
- 需要频繁阻塞、显式批准、完整审计、交接或复盘。
- 低质量模型、新手用户、实习生或团队训练场景。

## 降级到 zero/fast

任务过小且无真实决策时，不要为了协议制造复杂度：

- typo、文案、单点配置、无风险日志补充可走 `zero`。
- 单文件/小范围但仍需要确认目标和验证时走 `fast`。
